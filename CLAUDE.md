## What This Is

A C# library implementing the [SPIFFE](https://spiffe.io/) standard for workload identity. It provides an API to fetch X.509 and JWT SVIDs (workload identity credentials) and trust bundles from a SPIFFE Workload API endpoint. It mirrors the design of the Go [go-spiffe](https://github.com/spiffe/go-spiffe) library.

## Architecture

### Core Flow

A workload uses `WorkloadApiClient` to connect to the SPIFFE Workload API over gRPC (via Unix socket, Windows named pipe, or HTTP). The client fetches or streams SVIDs and bundles. `X509Source`, `JwtSource`, and `BundleSource` wrap the client and provide thread-safe access to the latest credentials with automatic background refresh.

### Key Components

**`WorkloadApi/`** — The main abstraction layer:
- `IWorkloadApiClient` — Interface for fetching/watching X.509 SVIDs, JWT SVIDs, and bundles
- `WorkloadApiClient` — gRPC-based implementation; connects to `SPIFFE_ENDPOINT_SOCKET` env var by default
- `X509Source` — Thread-safe source of X.509 SVIDs + bundles; auto-updates via Watch stream; implements both `IX509SvidSource` and `IX509BundleSource`
- `JwtSource` / `BundleSource` — Analogous sources for JWT SVIDs and bundle-only use cases
- `GrpcChannelFactory` — Platform-specific channel creation (Unix socket, Windows named pipe, HTTP)

**`Svid/`** — Credential types:
- `X509Svid` — SPIFFE ID + X.509 certificate chain + private key
- `JwtSvid` — SPIFFE ID + JWT token + audience/expiry/claims; includes `ParseInsecure()` and `Parse()` (with bundle verification)

**`Bundle/`** — Trust anchors:
- `X509Bundle` / `X509BundleSet` — Trusted X.509 CA certs per trust domain
- `JwtBundle` / `JwtBundleSet` — JWT public keys per trust domain

**`Id/`** — Identity primitives:
- `SpiffeId` — Parsed SPIFFE URI (e.g., `spiffe://example.org/service`); validates per spec
- `TrustDomain` — The authority portion of a SPIFFE ID

**`Ssl/`** — TLS/mTLS for gRPC and ASP.NET Core:
- `SpiffeSslConfig` — Factory for `GrpcChannelOptions` / `SslServerAuthenticationOptions` with SPIFFE-aware peer validation
- `Authorizers` — Built-in authorization strategies: `AuthorizeAny`, `AuthorizeId`, `AuthorizeOneOf`, `AuthorizeMemberOf`, `AuthorizeIf`

**`Grpc/`** — `GrpcChannelFactory` for connecting to the Workload API socket

### Source Pattern

`X509Source`, `JwtSource`, and `BundleSource` all extend an abstract `Source` base class that:
- Uses `ReaderWriterLockSlim` for thread-safe reads during background Watch updates
- Tracks initialization state; factory methods like `CreateAsync` wait for the first update before returning, and callers can check `IsInitialized` if needed
- Implements `IDisposable` — always dispose sources when done

### Proto

`src/Proto/workload.proto` defines the SPIFFE Workload API gRPC service. The generated code lives in `src/Spiffe/WorkloadApi/` (auto-generated by `Grpc.Tools` at build time).

### Packages

Central package versions are managed in `Directory.Packages.props`. The version string `<SpiffeVersion>` controls the NuGet package version. Dev versions have a `-dev` suffix.

## Test Structure

`tests/Spiffe.Tests/` mirrors `src/Spiffe/` folder-for-folder. Integration tests are in `tests/Spiffe.Tests/Integration/` and require a running SPIFFE Workload API endpoint. `tests/Spiffe.Tests.Server/` is a helper gRPC server used by integration tests.
